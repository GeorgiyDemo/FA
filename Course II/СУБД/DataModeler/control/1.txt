Сценарий проекта:  

Требуется разработать модель данных для системы управления базой данных небольшого отеля.

Отель небольшой и состоит из отдельных домов. У каждого дома есть название и цена за 1 ночь. В некоторых из домов есть кондиционер, телевизор и сейф.
В отеле есть персонал, который отвечает за обслуживание дома. Один человек может обслуживать несколько домов.
У персонала есть имя, фамилия, должность, а также мобильный телефон для связи.

Конечно, у отеля есть клиенты. Система отеля хранит слндующую информацию о клиентах: Имя, фамилию, e-mail, контактный телефон и пожелания клиента, если такие имеются.
У каждого клиента есть документ, по которому он регистрируется в системе отеля. Каждый документ содержит название и исходный (бинарный) файл. В ряде случаем возможно добавление текстовой расшифровки документа и комментариев сотрудника.
В каждом доме есть холодильник, из которого клиент может брать различные напитки, стоимость которых также учитывается в заказах клиента. У каждого холодильника есть идентфикатор и название модели.
У каждого напитка есть название, цена за 1 шт и количество, которое доступно в холодильнике.

Заказы клиента. В заказе клиента всегда есть объект заказа и идентификатор клиента, которому этот заказ и принадлежит. Также есть количество заказа (в случае с напитками из холодильника - это их количество, в случае домов - кол-во ночей)
